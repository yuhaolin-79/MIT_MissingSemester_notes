
## 一、Shell 基础认知
### 1. 什么是 Shell
Shell 是**文本命令行交互解释器**，区别于图形化界面、语音助手，不受预设功能限制，自由度极高。
主流终端解释器：
- Bash：通用性最强，课程核心教学对象
- Zsh/Fish：体验更优，语法逻辑与Bash相通

### 2. Shell 核心优势
1. 操作效率远高于鼠标点击
2. 支持命令自由组合，批量自动化处理重复任务
3. 开源项目部署、服务运维、程序调试必备工具
4. 可编写脚本实现复杂流程自动化

### 3. 终端启动方式
| 系统 | 启动方式 |
| ---- | ---- |
| Linux | `Ctrl+Alt+T` |
| Windows | 启动WSL2终端 |
| macOS | `Cmd+空格` 聚焦搜索 Terminal |

## 二、Shell 基础操作
### 1. 基础执行命令
```bash
# 查看系统时间
date
# 打印输出内容
echo hello
```
- Shell 默认以**空格**分割命令与参数
- 含空格/特殊字符路径：使用单/双引号包裹，或`\`转义

### 2. 万能帮助命令
```bash
# 查看命令完整官方手册
man 命令名
# 快速精简帮助（绝大多数命令通用）
命令名 --help
```

### 3. 目录导航核心命令
1. **cd 切换目录**（Shell内置命令，非独立程序）
```bash
cd /bin    # 绝对路径跳转
cd ~       # 跳转用户家目录
cd ..      # 跳转上级目录
cd .       # 当前目录
cd /       # 跳转系统根目录
```
2. **pwd 查看当前所在目录**
```bash
pwd
echo $PWD
```

### 4. 绝对路径 & 相对路径
- 绝对路径：以`/`开头，从系统根目录完整定位
- 相对路径：不以`/`开头，基于**当前目录**定位

## 三、系统可执行程序查找原理
### 1. PATH 环境变量
Shell 执行命令时，会遍历`$PATH`内所有目录，匹配对应可执行文件
```bash
# 查看全局命令搜索路径
echo $PATH
# 定位命令真实存放路径
which 命令名
```
系统所有可用命令，均存放于`$PATH`包含的`/bin`、`/usr/bin`等目录中。

### 2. 高频基础文件工具
| 命令 | 功能 |
| ---- | ---- |
| `cat` | 读取打印文件全部内容 |
| `sort` | 文本内容行排序 |
| `uniq` | 去除连续重复行 |
| `head` | 查看文件头部内容 |
| `tail` | 查看文件尾部内容 |
| `grep` | 文本内容关键词匹配搜索，支持正则 |

### 3. 进阶三大利器
#### ① sed 流式文本编辑
多用于批量文本替换
```bash
sed -i 's/原内容/新内容/g' 文件名
```
- `-i`：直接修改原文件
- `s`：替换语法标识
- `g`：全行全局替换

#### ② find 递归文件查找
**语法**：`find 查找路径 筛选条件`
```bash
# 查找下载目录30天前的zip压缩包
find ~/Downloads -type f -name "*.zip" -mtime +30
# 查找家目录100M以上大文件
find ~ -type f -size +100M -exec ls -lh {} \;
# 查找含TODO注释的Python文件
find . -name "*.py" -exec grep -l "TODO" {} \;
```
常用参数：
- `-type f`：仅匹配文件；`-type d`：仅匹配文件夹
- `-name`：按文件名匹配
- `-exec`：找到文件后执行指定命令

### find 命令极简笔记
#### 基础格式
```bash
find 查找路径 筛选条件 -exec 执行命令 {} \;
```
- `{}` 代表找到的文件
- `\;` 结束执行语句

##### 1.按文件名查找
```bash
# 查找当前所有md文件并查看详情
find . -name "*.md" -exec ls -lh {} \;

# 忽略大小写查找
find . -iname "*.txt" -exec cat {} \;

# 查找指定名字文件
find . -name "test.sh" -exec echo 找到文件：{} \;
```

##### 2.按文件类型
```bash
# 只找文件
find . -type f -name "*.java"

# 只找文件夹
find . -type d -name "src"

# 查找空文件
find . -type f -empty
```

##### 3.按文件大小
```bash
# 大于100M
find . -size +100M -exec du -sh {} \;

# 小于10K
find . -size -10k
```

##### 4.按修改时间
```bash
# 1天内修改过
find . -mtime -1

# 30天前修改的
find . -mtime +30
```

##### 5.批量常用操作
```bash
# 批量删除后缀tmp文件
find . -name "*.tmp" -exec rm {} \;

# 批量复制文件
find . -name "*.jpg" -exec cp {} ./pic/ \;

# 批量查看文件行数
find . -name "*.py" -exec wc -l {} \;

# 批量查找含指定内容的文件
find . -name "*.md" -exec grep "笔记" {} \;
```

##### 6.实用小参数
| 参数        | 作用      |
| --------- | ------- |
| `-name`   | 精准匹配文件名 |
| `-iname`  | 忽略大小写   |
| `-type f` | 普通文件    |
| `-type d` | 文件夹     |
| `-exec`   | 对结果执行命令 |

##### 7.兼容带空格文件名（推荐）
```bash
find . -name "*.sh" -print0 | xargs -0 wc -l
```

>*以上是`find`的部分*

#### ③ awk 文本数据解析
擅长按**列**拆分解析格式化文本
```bash
# 打印文件第二列内容
awk '{print $2}' 文件名
# 逗号分隔文件，打印第二列
awk -F, '{print $2}' 文件名
```

## 四、Shell 脚本编程基础
### 1. 管道符 `|`
将前一个命令**标准输出**，作为后一个命令**标准输入**，串联多命令完成复杂处理。
课程实战示例：远程服务器统计高频SSH登录用户名
```bash
ssh 服务器地址 'journalctl -u sshd -b-1 | grep "Disconnected from"' \
  | sed -E 's/.*Disconnected from .* user (.*) [^ ]+ port.*/\1/' \
  | sort | uniq -c \
  | sort -nk1,1 | tail -n10 \
  | awk '{print $2}' | paste -sd,
```

### 2. 重定向
```bash
# 标准输出写入文件（覆盖）
命令 > 文件名
# 标准输出追加写入文件
命令 >> 文件名
# 文件内容作为命令标准输入
命令 < 文件名
```
三大标准流：
- 0：标准输入 stdin
- 1：标准输出 stdout
- 2：标准错误 stderr

### 3. 流程控制
1. 条件判断 `if`
2. 循环语句 `while` / `for`
3. 状态判断：`$?` 获取上一条命令执行状态，`0`代表执行成功

### 4. Shell 脚本规范
1. 脚本开头声明解释器（Shebang）
```bash
#!/bin/bash
```
2. 严格模式配置（规避脚本坑点）
```bash
set -euo pipefail
```
    - `-e`：命令执行失败直接退出脚本
    - `-u`：禁止使用未定义变量
    - `-o pipefail`：管道命令执行失败整体报错
3. 后台运行程序：命令末尾加 `&`

### 5. 脚本执行权限
编写完成的`.sh`脚本，需赋予执行权限才可直接运行
```bash
chmod +x 脚本名.sh
./脚本名.sh
```

## 七、课后练习

### 1.`ls -l`输出前10个字符的权限含义


格式：**第1位 = 文件类型** + **后面9位 = 权限（3组×3位）**

#### 10位拆分
```
第1位：类型
第2-4位：所有者权限（user）
第5-7位：同组权限（group）
第8-10位：其他人权限（other）
```

#### 权限字母含义
- `r` = 读
- `w` = 写
- `x` = 执行
- `-` = 无权限

```bash
drwxrwxrwx 1 fish fish     4096 Apr  4 16:38  AppData
lrwxrwxrwx 1 fish fish       34 Jul 27  2025 'Application Data' -> /mnt/c/Users/36259/AppData/Roaming
drwxrwxrwx 1 fish fish     4096 Jul 27  2025  Contacts
```
#### 例子1
```
drwxrwxrwx 1 fish fish 4096 Apr  4 16:38 AppData
```
前10位：**`d r w x r w x r w x`**

- **d**：目录（文件夹）
- **rwx**：所有者（fish）可读、可写、可执行
- **rwx**：同组用户可读、可写、可执行
- **rwx**：所有人都可读、可写、可执行
👉 意思：**完全开放的文件夹**

#### 例子2
```
lrwxrwxrwx 1 fish fish 34 Jul 27 2025 'Application Data' -> ...
```
前10位：**`l r w x r w x r w x`**

- **l**：链接（快捷方式）
- 后面全是 rwx：**完全开放的快捷方式**

### 2.理解通配符`glob`用法：`*.txt`、`file?.txt`

我用**最简单、最实用**的方式给你讲清楚 `glob` 通配符，就拿你说的 `*.txt`、`file?.txt` 举例，一看就会！

#### glob 是什么？
glob 就是**匹配文件名的通配符**，用来批量找文件，不用一个个写名字。
最常用 2 个符号：
- `*` = 匹配**任意长度**的字符（包括空）
- `?` = 匹配**任意 1 个**字符（必须有一个）

---

#### 1. `*.txt`
匹配 **所有以 .txt 结尾的文件**
- 能匹配：
  - `a.txt`
  - `123.txt`
  - `report.txt`
  - `笔记.txt`
- 不能匹配：
  - `file.jpg`
  - `txt`（没有点）

意思：**找所有 txt 文件**

---

#### 2. `file?.txt`
匹配 **`file` 后面跟 1 个任意字符，再跟 .txt**
- 能匹配：
  - `file1.txt`
  - `fileA.txt`
  - `file9.txt`
  - `file_.txt`
- 不能匹配：
  - `file.txt`（少了一个字符）
  - `file10.txt`（多了字符）
  - `fileAA.txt`

意思：**找 file 后面只有 1 个字符的 txt 文件**

### 3.区分单引号、双引号、`$''` ANSI引号差异

1. **单引号 `' '`**：最强屏蔽，**啥都不解析**，原封不动
2. **双引号 `" "`**：**解析变量、转义**，但保留大部分原样
3. **$' '**：**ANSI 引号**，专门解析 **\n \t \r** 这类转义字符

---

#### 1. 单引号 `' '` —— 纯文本，不解析任何东西
**特点：**
- 里面的 **$变量、\转义、`命令` 全都不生效**
- 最安全，输出完全和你写的一样

例子：
```bash
name=张三
echo '$name  \n  不会换行'
```
输出：
```
$name  \n  不会换行
```

---

#### 2. 双引号 `" "` —— 解析变量，保留空格
**特点：**
- 会解析 **$变量**
- 会解析 **\" \\ \$** 少量转义
- **不会解析 \n 换行**

例子：
```bash
name=张三
echo "$name  \n  不换行"
```
输出：
```
张三  \n  不换行
```

---

#### 3. $' ' —— ANSI 引号（专门用来换行、制表）
**特点：**
- 能解析 **\n 换行、\t 制表符、\r 回车** 等 ANSI 转义

例子：
```bash
echo $'第一行\n第二行\t制表符'
```
输出：
```
第一行
第二行    制表符
```

```bash
fish@laptop:/mnt/c/Users/36259$ echo $'first line\nsecond line\n\ttable'
first line
second line
        table
```

| 引号类型 | 解析 $变量 | 解析 \n 换行 | 用途 |
|--------|-----------|-------------|------|
| `' '`  | ❌ 不解析 | ❌ 不解析 | 纯文本、固定字符串 |
| `" "`  | ✅ 解析 | ❌ 不解析 | 带变量的字符串 |
| `$' '` | ❌ 不解析 | ✅ 解析 | 需要换行、制表符 |

### 4.实现标准输出、错误输出分离重定向

文件描述符：
- `1`：标准输出 stdout（默认屏幕）
- `2`：标准错误 stderr（默认屏幕）

#### 1. 分离重定向（最常用）
```bash
# 正确输出进 out.log，错误输出进 err.log
cmd 1>out.log 2>err.log
```
简写：`1>` 可省略为 `>`
```bash
cmd > out.log 2> err.log
```

#### 2. 只丢错误，正常输出屏幕
```bash
cmd 2> err.log
```

#### 3. 只丢正常输出，错误留屏幕
```bash
cmd > out.log
```

#### 4. 错误输出合并到标准输出（合并重定向）
```bash
# 错误并入标准输出，一起写入文件
cmd > all.log 2>&1
```

#### 5. 丢弃输出 / 丢弃错误
```bash
# 丢弃正常输出
cmd > /dev/null
# 丢弃错误输出
cmd 2> /dev/null
# 全部丢弃
cmd > /dev/null 2>&1
```

#### 示例实操
```bash
# 查找文件，正确结果存 ok.txt，错误信息存 fail.txt
find / -name *.txt > ok.txt 2> fail.txt
```
